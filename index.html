Arquillian 1.0.2 has arrived!

Don't write another test until you study this one ▶

  1. Delegates execution of the test to Arquillian, which runs the test inside the active container
  2. Creates an archive to isolate classes and resources for the test scenario
  3. Wires the component from the archive to the test case
  4. Invokes a method on the component and validates its return value
@RunWith(Arquillian.class) // (1)
public class GreeterTest {
    @Deployment // (2)
    public static JavaArchive createDeployment() {
        return ShrinkWrap.create(JavaArchive.class)
            .addClass(Greeter.class)
            .addAsManifestResource(EmptyAsset.INSTANCE, "beans.xml");
    }

    @Inject // (3)
    Greeter greeter;

    @Test // (4)
    public void should_create_greeting() {
        assertEquals("Hello, Earthling!", greeter.greet("Earthling"));
    }
}

Featured Noble

  • Karel Piwko

    Brno, Czech Republic

    Karel Piwko is a member of the Arquillian and ShrinkWrap core teams. He works at Red Hat as a QA Associate in the JBoss QE group.

    He’s recognized here for his contribution of Arquillian Drone, an extension that simplifies the task of writing functional tests by managing the lifecycle of browser automation tools such as Selenium, WebDriver and Arquillian Graphene. Karel is now lead of the Drone extension.

    In addition to Drone, Karel helps design and implement ShrinkWrap Resolvers, an API that plays a key role in Drone-based functional tests.
